# Code of Conduct

## Our Standard

Volum welcomes contributors with different levels of experience. Project
discussions should make it easier for people to understand the problem, learn
from feedback, and improve the software safely.

Expected behavior includes:

- Be respectful, direct, and specific.
- Critique code and decisions, not people.
- Explain technical reasoning and define unfamiliar terms.
- Assume good intent while still addressing unsafe or harmful behavior.
- Make review feedback actionable and distinguish required changes from
  optional suggestions.
- Give credit for ideas, investigation, code, design, and documentation.
- Protect private information found in logs, screenshots, and test data.

Unacceptable behavior includes:

- Harassment, discrimination, threats, or personal attacks
- Sexualized language or unwanted attention
- Deliberate intimidation, disruption, or repeated off-topic contact
- Publishing another person's private information without permission
- Dismissing or humiliating people because of experience level
- Retaliation against someone who reports a concern in good faith

## Scope

This policy applies to issues, pull requests, reviews, discussions, release
work, and other spaces where someone represents the Volum project.

## Reporting

For an immediate public problem, report the comment or account through GitHub
and avoid escalating the exchange.

For a private report, contact the repository owner using the private contact
method published on the
[maintainer's GitHub profile](https://github.com/shirishkoirala). Include:

- Where the behavior occurred
- A concise description
- Relevant links or screenshots
- Any immediate safety concern

Do not open a public issue containing private or sensitive details.

## Enforcement

Project maintainers may edit or remove content and may warn, temporarily
restrict, or permanently exclude participants whose behavior violates this
policy. Reports will be reviewed as privately and promptly as practical.

Maintainers who have a conflict of interest should not make the enforcement
decision for that report.
